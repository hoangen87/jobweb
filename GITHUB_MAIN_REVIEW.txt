CONFIRMED ISSUES

1. CRITICAL - Next.js 14.2.15 has a known App Router DoS vulnerability.
   File: package.json, line 16
   The application uses the App Router. Next.js' official advisory identifies
   14.2.35 as the fixed 14.x release.
   Reference: https://nextjs.org/blog/security-update-2025-12-11

2. HIGH - Production build mutates the production database schema.
   File: package.json, line 7
   The build script runs `prisma db push` on every deployment. This changes the
   schema without a versioned migration history. It can stop deployment on a
   destructive change, or require accepting potential data loss.
   Reference: https://www.prisma.io/docs/orm/more/best-practices

3. HIGH (if Vercel lacks SESSION_SECRET) - Public fallback signing secret.
   File: src/lib/session.ts, line 11
   If SESSION_SECRET is not configured, session tokens are signed with the
   publicly visible string "fallback-secret", allowing forged admin cookies.

4. HIGH (if seed runs without environment variables) - Public default admin
   credentials.
   File: prisma/seed.ts, lines 59-60
   The seed defaults to username `admin` and password `Jhonsin@2026`.

5. MEDIUM - English/Chinese job search does not search translated job titles.
   File: src/app/[locale]/page.tsx, lines 31 and 42
   Database filtering searches the Vietnamese `title` first. Translation is
   only applied after filtering, so an English/Chinese query cannot match a
   translated title.

6. MEDIUM - English/Chinese pages format salary and dates in Vietnamese.
   File: src/lib/format.ts, lines 1-11
   Salary labels and date locale are hard-coded to Vietnamese and are used by
   JobCard and the job detail page.

7. MEDIUM - The HTML document language is always Vietnamese.
   File: src/app/layout.tsx, line 5
   Even /en and /zh render `<html lang="vi">`, affecting accessibility and SEO.
